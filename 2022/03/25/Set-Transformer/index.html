Set Transformer

Set Transformer

1.Abstract

本文针对无序的一组输入提出了一种基于注意力的模型，以此减少点积的运算，将运算复杂度从平方减少到了线性。

2. Introduction

深度学习解决的大部分问题是个例输入问题(instance-input)，将一个固定维度的输入映射到一个目标值上；而集合输入问题(set-input)具有两个条件：

  • 排列不变性，即当输入集合的元素顺序改变时，输出不应该发生改变。
  • 输入集合大小无限制。

Set Transformer主要有以下优点：

  • 利用自注意力机制发掘输入集合里元素两两或更多之间的关联。
  • 将自注意力机制的$O(n^2)$的计算复杂度降到了$O(nm)$，其中$m$是一个固定的超参数。
  • 利用子注意力机制进行特征聚合（以往的工作里常用的是各种池化操作），在需要多输出的问题里很有用。

3.模型结构

3.1排列不变集合注意力块

不同于以往工作的池化操作来独立提取集合中每个元素的特征，本文的注意力块采用Transformer中提出的多注意力机制，同时对输入进行encoding，以此提取输入间的联系。首先定义以下几个单元：

  • MAB(Multihead Attention Block),即Transformer中的多头注意力去掉position encoding：

    Figure1. MAB

  • 在MAB基础上定义SAB(Set Attention Block):

    $$ SAB(X):=MAB(X,X) $$
    Figure2. SAB

利用SAB可以提取到输入两两之间的关系，因此可以通过堆叠SAB来提取更多个输入之间的关系。但因为SAB的计算复杂度是$O(n^2)$，在堆叠过程中可能会有很高的计算代价，因此本文又提出了ISAB.

  • ISAB(Induced Set Attention Block)：将SAB中的query改成一个可学习的矩阵$I\epsilon R^{m*d}$，$I$被称为诱导点，其首先和$X$经过MAB提取到输入间的信息得到$H\epsilon R^{m*d}$，H再和X经过MAB得到输出。作者希望用$I$学到输入的全局特征。
Figure3. ISAB

可以看出，ISAB把运算复杂度降到了$O(nm)$，并且由于多头注意力的输出是输入的加权线性组合，因此SAB和ISAB均具有排列不变性。

3.2多头注意力池化

将特征$ Z\epsilon R^{n*d} $提取到以后，以往的工作在进行特征聚合的时候是通过平均池化或相加池化，这是建立在特征的权重是相等的基础上的。Set Transformer利用一个可学习的矩阵$S\epsilon R^{k*d}$来进行不同权重的特征聚合：
$$ PMA_k(Z)=MAB(S,rFF(Z)) $$
$rFF$就是普通的前向传递，并且根据实验（魔法来咯）在代价聚合后对其结果再进行自注意力会得到更好的结果：
$$ H=SAB(PMA_k(Z)) $$

3.3结构总览

Set Transformer的结构由编码器和解码器组成，编码器就是SAB或者ISAB的堆叠，ISAB拥有更低的运算复杂度；解码器就是对编码器得到的特征进行聚合后再经由自注意力得到输出。
